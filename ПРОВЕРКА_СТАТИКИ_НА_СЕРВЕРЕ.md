# Проверка и исправление статики на сервере

## Проблема
Статика работала при `DEBUG=True` для `onesim8n.beget.tech`, но перестала работать при `DEBUG=False` для `onesimus25.ru`.

## Решение

### Шаг 1: Убедитесь, что на сервере обновленный `config/urls.py`

На сервере проверьте, что в `config/urls.py` НЕТ условия `if settings.DEBUG:`:

```bash
cd ~/onesimus/onesimus
grep -n "if.*DEBUG" config/urls.py
```

Если команда ничего не выводит - хорошо. Если выводит строки с `if settings.DEBUG:` или `if DEBUG:` - нужно обновить файл.

### Шаг 2: Загрузите обновленный `config/urls.py` на сервер

Файл уже исправлен локально. Загрузите его на сервер через:
- Git: `git add config/urls.py && git commit -m "Fix static files serving" && git push`
- Или через файловый менеджер Beget

### Шаг 3: Проверьте, что Django может раздать статику

На сервере выполните:

```bash
cd ~/onesimus/onesimus
source venv/bin/activate

# Проверьте, что файлы существуют
ls -la staticfiles/css/style.css
ls -la static/css/style.css

# Перезапустите Django
touch tmp/restart.txt

# Подождите 10-20 секунд и проверьте
curl -I http://localhost:8000/static/css/style.css 2>/dev/null || \
curl -I https://onesimus25.ru/static/css/style.css
```

### Шаг 4: Если Nginx перехватывает запросы

Если `curl` возвращает 404, возможно Nginx на Beget автоматически перехватывает `/static/` для нового домена.

**Вариант А: Проверьте через прямой доступ к Django (минуя Nginx)**

На сервере временно запустите Django на другом порту:

```bash
cd ~/onesimus/onesimus
source venv/bin/activate
python manage.py runserver 127.0.0.1:8001
```

В другом терминале проверьте:

```bash
curl -I http://127.0.0.1:8001/static/css/style.css
```

Если возвращает 200 - Django работает правильно, проблема в Nginx.

**Вариант Б: Создайте тестовый view для проверки**

Добавьте в `config/urls.py` перед другими URL:

```python
from django.http import FileResponse
from django.conf import settings
import os

def test_static(request):
    """Тестовый endpoint для проверки статики"""
    static_file = os.path.join(settings.STATIC_ROOT, 'css', 'style.css')
    if os.path.exists(static_file):
        return FileResponse(open(static_file, 'rb'), content_type='text/css')
    return HttpResponse(f"File not found: {static_file}", status=404)
```

И добавьте в `urlpatterns`:

```python
path('test-static/', test_static, name='test_static'),
```

Проверьте: `curl -I https://onesimus25.ru/test-static/`

### Шаг 5: Временное решение - верните DEBUG=True

Если ничего не помогает, временно верните `DEBUG=True` в `config/settings.py` на сервере:

```bash
sed -i "s/DEBUG = False/DEBUG = True/" config/settings.py
touch tmp/restart.txt
```

Это вернет работу статики, но не рекомендуется для production.

## Проверка после исправления

```bash
# Проверьте через curl
curl -I https://onesimus25.ru/static/css/style.css

# Должен вернуться:
# HTTP/2 200
# Content-Type: text/css
```

Если возвращается 404, значит Nginx перехватывает запросы. В этом случае нужно настроить Nginx через панель Beget или обратиться в поддержку.
