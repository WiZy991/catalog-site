Техническое задание на разработку сайта
1. Общая информация
Тип проекта: веб-сайт каталога товаров с возможностью автоматического и ручного обмена данными между 1С, сайтом и площадкой Farpost.
Цель для покупателя: предоставить удобный онлайн-каталог с товарами, фильтрами и карточками, а также возможность перехода на магазин клиента на Farpost для совершения покупки.
Цель для продавца: создать центральный элемент системы автоматизации продаж, обеспечивающий массовое и синхронное размещение товаров на нескольких ресурсах:
1С → Сайт → Farpost, включая:
загрузку прайса из 1С,
автоматическое создание карточек товаров на сайте,
привязку изображений,
автоматическую передачу данных обратно в Farpost в расширенном формате.

2. Основные задачи проекта
Создать веб-сайт с простой структурой по аналогии с референсами:
https://onesimus25.ru/
https://autoroom25.ru/
Реализовать каталог товаров с разделами, подкатегориями и фильтрами.
Для каждого товара реализовать карточку, содержащую:
название;
основное изображение;
цену;
краткое описание (артикул, бренд, наличие, характеристика, применимость, кросс-номер);
кнопку «Перейти», ведущую на карточку товара на Farpost.
Создать CMS с удобным управлением:
товарами,
карточками и характеристиками,
категориями,
изображениями,
ценами,
ссылками.
Обеспечить автоматический и ручной обмен товарными данными между:
1С → Сайт,
Сайт → Farpost.
Сайт должен уметь формировать прайс-лист Farpost, включая:
ссылки на изображения,
структурированные характеристики,
уникальные ссылки на карточки товара на сайте.

3. Функциональные требования
3.1. Главная страница
Краткое описание компании.
Визуальные блоки:
преимущества,
акции,
категории каталога.
Быстрый переход в каталог.

3.2. Каталог товаров
3.2.1. Структура каталога
Категории и подкатегории (пример: Авто → Колеса → Шины).
Неограниченная вложенность разделов.
Возможность свободного добавления новых разделов через CMS.
3.2.2. Фильтры
Фильтрация по параметрам товара:
цена;
бренд;
размер;
состояние (новое/б/у);
применимость;
характеристики;
кросс-номер;
наличие.
Фильтры должны быть комбинируемыми и работать без перезагрузки страницы (желательно AJAX).
3.2.3. Вид каталога (список)
Каждая карточка товара в списке должна включать:
миниатюру изображения,
название,
цену,
краткие параметры (артикул, бренд, наличие, характеристика, применимость, кросс-номер),
кнопку «Подробнее».

3.3. Страница карточки товара
На карточке должны быть:
большое изображение + фотогалерея;
название;
цена;
расширенное описание;
технические характеристики;
контактные данные (по необходимости);
кнопка «Перейти», ведущая на соответствующее объявление Farpost.
3.3.1. Кнопка «Перейти»
визуально заметная,
открывает Farpost в новой вкладке,
ссылка индивидуальная для каждого товара.

4. Административная панель (CMS)
CMS должна включать:
4.1. Управление товарами
добавление / удаление / редактирование товаров,
редактирование характеристик,
управление ценами,
загрузка фотографий,
ручное добавление ссылки на Farpost (при необходимости).
4.2. Управление категориями
создание, удаление и изменение категорий,
группировка товаров,
массовое перемещение позиций между разделами.
4.3. Импорт/экспорт
Ручной импорт прайса:
загрузка XLS/CSV,
сопоставление столбцов,
массовое создание карточек.
Экспорт прайса:
формирование расширенного прайса для Farpost,
включение:
сведений о товаре,
характеристик,
цен,
наличия,
ссылок на сайт,
ссылок на изображения.

5. Автоматический обмен данными
Главное требование: автоматическая синхронизация:
5.1. 1С → Сайт
автоматический прием прайса из 1С,
создание и обновление товаров,
обновление характеристик, цен, наличия,
безопасный формат обмена (CSV/XLS/XML/API — определяется отдельно).
5.2. Сайт → Farpost
Сайт должен:
формировать расширенный прайс, понятный Farpost,
включать ссылки на изображения с сайта,
структурированные характеристики,
формировать корректные заголовки,
автоматически отправлять данные на Farpost.
Публикация должна осуществляться массово и без ручного редактирования.

6. Логика работы изображений
Изображения хранятся на сайте,
Для каждой карточки можно загружать одну или несколько фотографий,
Система автоматически генерирует прямые ссылки,
Эти ссылки вписываются в прайс для Farpost.

7. Требования к структуре объявлений Farpost
Каждое объявление Farpost должно содержать:
товарный заголовок,
фото,
цену,
характеристики,
описание,
применимость,
кросс-номер,
артикул и бренд.
Заголовки и описание должны формироваться:
автоматически на основе данных из прайса,
по шаблонам, настраиваемым в CMS.
Объявления должны визуально и структурно соответствовать качеству конкурентов, например:
https://autoroom25.ru/cat/detail/starter_isuzu_10pd1_12pd1_24_volt/
https://www.farpost.ru/vladivostok/zapchasti/starter-isuzu-10pd1-12pd1-24-volt-g17573721205.html


8. Дизайн
адаптивный,
аккуратный,
современный,
без лишних визуальных эффектов,
логика расположения элементов аналогична сайтам-референсам,
каталог и карточки интуитивно читаемые.

9. Технические требования
наличие CMS обязательно,
сайт должен работать на любом современном хостинге,
поддержка мобильных устройств и планшетов,
защита персональных данных (при наличии форм обратной связи).


10. Тестирование перед запуском
Проверить:
корректность фильтров,
работу импорта/экспорта,
корректность генерации изображений,
успешный обмен с Farpost,
адаптивность и скорость загрузки,
корректность ссылок «Перейти».

11. Требования к надежности
логирование обменов,
обработка ошибок импорта,
уведомления о неудачах,
механизм отката данных (по договоренности),
12.SEO-требования (расширенная версия)
Сайт должен быть разработан с учётом всех современных требований поисковых систем и предусматривать дальнейшее развитие SEO-инфраструктуры.
12.1. Базовая SEO-оптимизация
ЧПУ-адреса
Формирование удобочитаемых URL:
site.ru/category/podcategory/tovar-artikul
Автоматическое транслитерирование русских названий.
Генерация мета-данных
Возможность редактирования Title, Description и H1 вручную.
Автоматическая генерация шаблонов для:
категорий,
подкатегорий,
карточек товаров (на основе артикулов, брендов, применимости).
SEO-тексты
Возможность добавлять текстовые блоки на:
главную страницу,
страницы категорий,
карточки товаров (при необходимости).
Микроразметка (Schema.org)
Обязательная поддержка:
Product,
Breadcrumbs,
Offer,
ImageObject.
Корректный вывод цены, наличия, фото товара.
XML-карта сайта
Автоматическая генерация sitemap.xml.
Регулярное обновление при изменениях на сайте.
robots.txt
Настраиваемый файл с запретом на индексацию служебных разделов.

12.2. Техническая SEO-оптимизация
Адаптивная вёрстка
Правильный mobile-friendly дизайн.
Отсутствие горизонтальной прокрутки.
Скорость загрузки
Оптимизация изображений.
Lazy load для фотографий в каталоге.
Минимизация CSS/JS.
Поддержка WebP (желательно).

Канонические ссылки
Для карточек с вариантами позиций.
Для страниц с параметрическими фильтрами.
Структура заголовков H1–H6
На каждой странице ровно один H1.
Корректная иерархия заголовков.
Пагинация
Реализация rel="next" и rel="prev".
Генерация ЧПУ-адресов для страниц каталога.
Фильтры
Возможность выбора:
индексируемые фильтры,
неиндексируемые фильтры.
Запрет индексации параметров, которые создают дубли.

12.3. SEO для каталога и карточек товаров
Уникальность контента
Возможность задавать уникальные описания.
Генерация короткого описания автоматически при импорте.

Структурирование данных
Характеристики должны выводиться в виде структурированного блока.
Автоматическая генерация заголовков
Пример шаблона:
Бренд + Артикул + Краткая характеристика
Например:
"Starter Isuzu 10PD1 / 12PD1, 24V — оригинал"
ALT-теги
Автоматическая подстановка alt по шаблону:
"Фото товара Артикул Бренд".
Внутренняя перелинковка
Ссылки:
на другие товары бренда,
на совместимые запчасти,
на разделы каталога.

12.4. Локальная SEO-оптимизация (при необходимости)
Если у компании есть адрес:
вывод контактной информации в формате микроразметки,
карта на сайте,
корректные NAP-данные,
отдельная страница «Контакты» с метками.

12.5. SEO интеграция с Farpost
Да, Farpost сам по себе не влияет на SEO, но структура карточек должна быть такой, чтобы:
товары с сайта корректно индексировались поисковиками,
карточки могли появляться в выдаче по:
артикулу,
бренду,
характеристикам,
применимости.
Поэтому важно:
формировать описание без дублей,
правильно заполнять заголовки,
избегать пустых страниц, если товар закончился.
