# Проверка настройки cron для автоматического импорта

## ✅ Файл `cron_1c_import.log` появится автоматически

Файл `logs/cron_1c_import.log` **создастся автоматически** при первом запуске cron задачи. Его отсутствие сейчас - это нормально!

## 🧪 Тестирование перед настройкой cron

### Шаг 1: Проверьте команду вручную

Через SSH выполните:

```bash
cd ~/onesimus/onesimus
source venv/bin/activate
python manage.py process_1c_files --recent 10
```

Если команда выполнилась без ошибок, значит всё настроено правильно.

### Шаг 2: Проверьте создание лог-файла

После выполнения команды проверьте, создался ли файл:

```bash
ls -la logs/cron_1c_import.log
```

Или через файловый менеджер Beget проверьте папку `logs/` - должен появиться файл `cron_1c_import.log`.

### Шаг 3: Проверьте содержимое лог-файла

```bash
cat logs/cron_1c_import.log
```

Должны увидеть вывод команды обработки файлов.

## 📋 Настройка cron через Beget

После того, как убедились, что команда работает:

1. Откройте **"Планировщик заданий"** в панели Beget
2. Используйте **"Мастер заданий"** или **"Составить задание вручную"**
3. Укажите команду:
   ```bash
   cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --recent 10 >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
   ```
4. Настройте расписание (например, каждые 5 минут: `*/5 * * * *`)
5. Сохраните задание

## 🔍 Проверка работы cron

### Через несколько минут после настройки:

1. Проверьте файл `logs/cron_1c_import.log` через файловый менеджер Beget
2. Файл должен появиться и содержать вывод команды
3. Если файл пустой или не появился - проверьте настройки cron

### Через SSH:

```bash
# Проверьте, что файл создался
ls -la logs/cron_1c_import.log

# Посмотрите содержимое
tail -f logs/cron_1c_import.log

# Или весь файл
cat logs/cron_1c_import.log
```

## ⚠️ Если файл не создается

### 1. Проверьте права на директорию logs

```bash
ls -la logs/
```

Должны увидеть что-то вроде:
```
drwxr-xr-x 2 onesim8n onesim8n 4096 Feb 18 06:39 .
```

Если права неправильные, исправьте:
```bash
chmod 755 logs
```

### 2. Проверьте, что команда работает

```bash
cd ~/onesimus/onesimus
source venv/bin/activate
python manage.py process_1c_files --recent 10 >> logs/cron_1c_import.log 2>&1
```

После этого проверьте:
```bash
cat logs/cron_1c_import.log
```

### 3. Проверьте настройки cron в Beget

- Убедитесь, что задача активна (включена)
- Проверьте правильность путей в команде
- Проверьте cron выражение (расписание)

### 4. Проверьте системные логи

Если в Beget есть доступ к системным логам, проверьте:
```bash
grep CRON /var/log/syslog
```

Или через панель Beget найдите раздел с логами cron.

## 📊 Альтернатива: без лог-файла

Если не хотите использовать отдельный лог-файл, можно убрать `>> ...` из команды:

```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --recent 10
```

Тогда вывод будет отправляться на email (если настроен в Beget) или просто теряться.

Но лучше оставить лог-файл для отладки!

Выполните эту команду через SSH, чтобы создать файл и проверить работу:

```bash

```

После этого проверьте файл через файловый менеджер Beget или через SSH:
```bash
cat logs/cron_1c_import.log
```
