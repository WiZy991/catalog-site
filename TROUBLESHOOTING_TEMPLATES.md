# Решение проблемы: изменения не отображаются после git pull

## Возможные причины

1. **Кэш браузера** - браузер показывает старую версию
2. **Python bytecode кэш** - старые .pyc файлы
3. **Неправильный перезапуск приложения**
4. **Файлы не обновились** - нужно проверить

## Решение

### 1. Проверьте, что файлы действительно обновились

```bash
# Проверьте дату изменения файла
ls -lh templates/partners/catalog.html

# Проверьте содержимое файла (должны быть новые изменения)
grep -n "category-list__toggle" templates/partners/catalog.html
```

### 2. Очистите Python bytecode кэш

```bash
# Найти и удалить все .pyc файлы
find . -type d -name __pycache__ -exec rm -r {} + 2>/dev/null || true
find . -name "*.pyc" -delete
find . -name "*.pyo" -delete
```

### 3. Правильно перезапустите приложение

```bash
# Для Beget (если используется)
touch ../tmp/restart.txt

# Или перезапустите через systemd (если используется)
# sudo systemctl restart gunicorn
# sudo systemctl restart uwsgi
```

### 4. Очистите кэш Django (если используется)

```bash
python manage.py shell
>>> from django.core.cache import cache
>>> cache.clear()
>>> exit()
```

### 5. Проверьте в браузере

1. **Жёсткая перезагрузка**: `Ctrl + Shift + R` (Linux/Windows) или `Cmd + Shift + R` (Mac)
2. **Очистите кэш браузера**: Откройте DevTools (F12) → Network → включите "Disable cache"
3. **Проверьте в режиме инкогнито**

### 6. Проверьте логи приложения

```bash
# Проверьте логи на наличие ошибок
tail -f log_django.txt

# Или если используется другой логгер
tail -f /var/log/gunicorn/error.log
```

### 7. Проверьте, что изменения в правильном файле

Убедитесь, что вы смотрите правильный URL:
- Оптовая часть: `/partners/catalog/`
- Проверьте, что вы авторизованы как партнёр

### 8. Если ничего не помогает - полная перезагрузка

```bash
# 1. Очистите кэш
find . -type d -name __pycache__ -exec rm -r {} + 2>/dev/null || true
find . -name "*.pyc" -delete

# 2. Перезапустите приложение
touch ../tmp/restart.txt

# 3. Подождите 10-15 секунд

# 4. Проверьте в браузере с Ctrl+Shift+R
```

## Проверка изменений

После выполнения команд проверьте:

1. **Аккордеон категорий**: Должны быть стрелочки у категорий с подкатегориями
2. **Заголовки таблицы**: Не должны накладываться друг на друга
3. **JavaScript**: Должен работать клик по стрелкам для раскрытия подкатегорий

## Если проблема остаётся

1. Проверьте, что вы на правильной странице (оптовая часть, не розничная)
2. Проверьте, что вы авторизованы как партнёр
3. Проверьте консоль браузера на наличие JavaScript ошибок (F12 → Console)
4. Проверьте Network вкладку - загружается ли правильный HTML
