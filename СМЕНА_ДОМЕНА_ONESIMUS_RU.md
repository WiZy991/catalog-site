# Инструкция по смене домена на onesimus.ru

## 📋 Что нужно сделать

### 1. Настройка DNS для домена onesimus.ru

Вам нужно настроить DNS записи для домена `onesimus.ru`:

**Вариант A: Если домен на Beget**
1. Зайдите в панель управления Beget
2. Перейдите в раздел "Домены" → "Управление DNS"
3. Найдите домен `onesimus.ru`
4. Убедитесь, что есть A-запись, указывающая на IP сервера Beget
5. Или используйте CNAME запись, указывающую на `onesim8n.beget.tech`

**Вариант B: Если домен на другом регистраторе**
1. Зайдите в панель управления вашего регистратора домена
2. Найдите настройки DNS для домена `onesimus.ru`
3. Добавьте A-запись:
   - Тип: A
   - Имя: @ (или оставьте пустым)
   - Значение: IP адрес сервера Beget (узнайте в панели Beget)
4. Добавьте CNAME запись для www:
   - Тип: CNAME
   - Имя: www
   - Значение: onesimus.ru

### 2. Настройка домена в панели Beget

1. Зайдите в панель управления Beget
2. Перейдите в раздел "Домены и поддомены"
3. Добавьте домен `onesimus.ru` к вашему аккаунту
4. Привяжите домен к существующему сайту (к тому же, где сейчас `onesim8n.beget.tech`)
5. Если Beget предложит выбрать директорию, укажите ту же, что используется для текущего сайта

### 3. Настройка SSL сертификата

После добавления домена в Beget:

1. В панели Beget найдите раздел "SSL сертификаты"
2. Выберите домен `onesimus.ru`
3. Установите бесплатный SSL сертификат Let's Encrypt (обычно это делается автоматически)
4. Дождитесь активации сертификата (может занять несколько минут)

### 4. Обновление настроек Django

✅ **Уже сделано!** В файле `config/settings.py` обновлен `ALLOWED_HOSTS`:
```python
ALLOWED_HOSTS = ['*', 'onesimus.ru', 'www.onesimus.ru', 'onesim8n.beget.tech']
```

Это означает, что Django будет принимать запросы с обоих доменов.

### 5. Проверка работы сайта

После настройки DNS и домена (может занять до 24 часов для распространения DNS):

1. Проверьте доступность сайта:
   ```bash
   curl -I http://onesimus.ru/
   curl -I https://onesimus.ru/
   ```

2. Проверьте работу CommerceML endpoint:
   ```bash
   curl -I https://onesimus.ru/cml/exchange?type=catalog&mode=checkauth
   ```

3. Откройте сайт в браузере:
   - http://onesimus.ru
   - https://onesimus.ru
   - http://www.onesimus.ru
   - https://www.onesimus.ru

### 6. Обмен с 1С и Farpost - ВАЖНО! ⚠️

**Хорошая новость:** Оба домена будут работать одновременно! Это означает, что:

#### ✅ Вариант 1: Оставить старый домен для обмена (РЕКОМЕНДУЕТСЯ)

**Преимущества:**
- Не нужно ничего менять в настройках 1С и Farpost
- Обмен продолжит работать без перерывов
- Новый домен `onesimus.ru` будет для пользователей сайта

**Что делать:**
- Просто добавьте новый домен `onesimus.ru` в Beget
- Старый домен `onesim8n.beget.tech` продолжит работать
- В настройках 1С и Farpost ничего менять не нужно

#### ✅ Вариант 2: Переключить на новый домен

Если хотите использовать новый домен для обмена:

**Обновление настроек в 1С:**

**Старый URL:**
```
http://onesim8n.beget.tech/cml/exchange?type=catalog&mode=checkauth
```

**Новый URL:**
```
https://onesimus.ru/cml/exchange?type=catalog&mode=checkauth
```

**Обновление настроек в Farpost:**

Если у вас настроен автоматический импорт прайс-листа в Farpost, обновите URL:

**Старый URL:**
```
http://onesim8n.beget.tech/farpost/price-list.csv
```

**Новый URL:**
```
https://onesimus.ru/farpost/price-list.csv
```

Или для других форматов:
- `https://onesimus.ru/farpost/price-list.xls`
- `https://onesimus.ru/farpost/price-list.xml`

#### 🔍 Как это работает технически

В коде Django все URL генерируются **динамически** через `request.build_absolute_uri()`. Это означает:

- Если запрос приходит с `onesim8n.beget.tech` → URL будут с `onesim8n.beget.tech`
- Если запрос приходит с `onesimus.ru` → URL будут с `onesimus.ru`

**Поэтому:**
- Оба домена работают автоматически
- Не нужно менять код
- Просто обновите настройки в 1С и Farpost, если хотите использовать новый домен

### 7. Настройка редиректа (опционально)

Если хотите, чтобы старый домен автоматически перенаправлял на новый, можно настроить редирект в панели Beget или через .htaccess:

**Через .htaccess:**
```apache
# Редирект с старого домена на новый
RewriteEngine On
RewriteCond %{HTTP_HOST} ^onesim8n\.beget\.tech$ [NC]
RewriteRule ^(.*)$ https://onesimus.ru/$1 [R=301,L]
```

**Или через панель Beget:**
- Найдите настройки редиректа для домена `onesim8n.beget.tech`
- Установите редирект на `https://onesimus.ru`

### 8. Обновление ссылок в коде (если есть)

Проверьте, нет ли в коде жестко прописанных ссылок на старый домен:

```bash
# Поиск ссылок на старый домен
grep -r "onesim8n.beget.tech" . --exclude-dir=venv --exclude-dir=.git
```

Если найдете, замените на новый домен или используйте относительные пути.

## ⚠️ Важные моменты

1. **DNS может распространяться до 24-48 часов** - будьте терпеливы

3. **SSL сертификат** - убедитесь, что он установлен для нового домена
4. **1С и Farpost обмен** - можно оставить старый домен для обмена или обновить на новый (см. раздел 6)
5. **Никаких изменений в коде не требуется** - все URL генерируются автоматически
