# Срочное исправление импорта товаров

## 🔴 Проблемы

1. **Миграция не применена** - ошибка `no such table: catalog_productcharacteristic`
2. **Cron использует `--recent 10`** - пропускает старые файлы
3. **Файлы уже помечены как обработанные** - не обрабатываются повторно

## ✅ Решение (пошагово)

### Шаг 1: Применить миграцию на сервере

Через SSH выполните:

```bash
cd /home/o/onesim8n/onesimus/onesimus
source venv/bin/activate
python manage.py migrate
```

Проверьте, что миграция применена:

```bash
python manage.py showmigrations catalog
```

Должна быть галочка `[X]` напротив `0012_synclog_productcharacteristic`.

### Шаг 2: Удалить маркеры обработанных файлов

Удалите маркеры `.processed`, чтобы файлы обработались снова:

```bash
cd /home/o/onesim8n/onesimus/onesimus/media/1c_exchange
rm -f *.processed
```

### Шаг 3: Исправить команду в cron

В Beget откройте "Планировщик заданий" и измените команду:

**Старая команда (неправильная):**
```bash
... process_1c_files --recent 10 >> ...
```

**Новая команда (правильная):**
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

**Важно:** Убрать `--recent 10` из команды!

### Шаг 4: Запустить импорт вручную для проверки

После применения миграции и удаления маркеров:

```bash
cd /home/o/onesim8n/onesimus/onesimus
source venv/bin/activate
python manage.py process_1c_files
```

Проверьте вывод - должно быть:
```
✓ import0_1.xml: обработано X товаров (создано: Y, обновлено: Z)
```

### Шаг 5: Проверить товары

1. Зайдите в админку Django
2. Откройте раздел "Товары"
3. Товары должны появиться

## 🔍 Проверка результата

После выполнения всех шагов:

1. **Проверьте миграции:**
   ```bash
   python manage.py showmigrations catalog | grep 0012
   ```
   Должно быть: `[X] 0012_synclog_productcharacteristic`

2. **Проверьте таблицу:**
   ```bash
   python manage.py dbshell
   ```
   Затем в SQLite:
   ```sql
   .tables | grep productcharacteristic
   ```
   Должна быть таблица `catalog_productcharacteristic`.

3. **Проверьте логи cron:**
   ```bash
   tail -20 logs/cron_1c_import.log
   ```
   Не должно быть ошибок `no such table`.

## ⚠️ Важно

- После применения миграции нужно перезапустить импорт
- Убедитесь, что в cron-задаче убран флаг `--recent 10`
- Маркеры `.processed` нужно удалить для повторной обработки файлов

## 📝 Быстрая команда для всего сразу

```bash
cd /home/o/onesim8n/onesimus/onesimus && \
source venv/bin/activate && \
python manage.py migrate && \
cd media/1c_exchange && \
rm -f *.processed && \
cd ../../ && \
python manage.py process_1c_files
```

Эта команда:
1. Применяет миграции
2. Удаляет маркеры
3. Запускает импорт
