# Подкатегории из ключевых слов (keywords)

## Что изменилось

Теперь система поддерживает **динамические подкатегории**, которые создаются из поля **`Ключевые слова`** в админке категории.

- **При сохранении категории в админке**: подкатегории синхронизируются сразу.
- **При обмене 1С (CommerceML)**: перед обработкой файла обмена подкатегории синхронизируются автоматически, чтобы изменения keywords “вступали в силу после обмена”.

## Формат поля `Ключевые слова`

Укажите слова **через запятую**:

Пример:

```
амортизатор, пружина, шаровая опора, рычаг
```

## Как создаются подкатегории

Для каждой фразы из `keywords` создаётся/обновляется дочерняя категория (подкатегория) под текущей категорией:

- Подкатегории создаются **только по ключевым словам с кириллицей** (чтобы не плодить `engine/piston/valve` и т.п.).
- **Имя подкатегории**: слово с заглавной буквой (например, `амортизатор` → `Амортизатор`)
- **keywords подкатегории**: выставляется равным этому слову (чтобы подкатегория автоматически находилась по названию товара)

Если слово удалили из `keywords`, соответствующая автосозданная подкатегория будет **деактивирована** (только если она выглядит как автосозданная: `keywords` совпадает с её именем).

## Как выбирается подкатегория для товара

Сначала проверяются **подкатегории из базы** (children с заполненными `keywords`), и если ключевое слово найдено в названии товара — товар попадает в эту подкатегорию.

Если совпадений по базе нет — используется старый fallback по хардкоду.
