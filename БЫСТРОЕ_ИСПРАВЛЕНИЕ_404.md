# БЫСТРОЕ ИСПРАВЛЕНИЕ: 404 для статики

## Проблема
- Файлы существуют: `staticfiles/css/style.css`, `staticfiles/js/main.js`
- Но `curl` возвращает `HTTP/2 404` с `content-type: text/html`
- Файл `.nginx/onesimus25.ru.conf` не существует

## Решение: Настроить nginx

### Вариант 1: Автоматическая настройка (рекомендуется)

```bash
cd ~/onesimus/onesimus

# Загрузите скрипт (если его еще нет)
# Или создайте его вручную (см. ниже)

# Запустите скрипт
bash setup_nginx_static.sh
```

### Вариант 2: Ручная настройка

```bash
cd ~/onesimus/onesimus

# 1. Создайте директорию
mkdir -p .nginx

# 2. Узнайте абсолютный путь к staticfiles
realpath staticfiles
# Запишите этот путь (например: /home/o/onesim8n/onesimus/onesimus/staticfiles)

# 3. Создайте конфигурацию
nano .nginx/onesimus25.ru.conf
```

Скопируйте в файл (замените путь на ваш):

```nginx
location /static/ {
    alias /home/o/onesim8n/onesimus/onesimus/staticfiles/;
    expires 30d;
    add_header Cache-Control "public, immutable";
    access_log off;
    
    types {
        text/css css;
        application/javascript js;
        image/png png;
        image/jpeg jpg jpeg;
        image/svg+xml svg;
        image/webp webp;
        font/woff2 woff2;
        font/woff woff;
    }
    default_type application/octet-stream;
}

location /media/ {
    alias /home/o/onesim8n/onesimus/onesimus/media/;
    expires 7d;
    add_header Cache-Control "public";
    access_log off;
}
```

**ВАЖНО**: Замените путь `/home/o/onesim8n/onesimus/onesimus/staticfiles/` на ваш реальный путь!

```bash
# 4. Создайте для www (если используется)
cp .nginx/onesimus25.ru.conf .nginx/www.onesimus25.ru.conf

# 5. Установите права
chmod 644 .nginx/onesimus25.ru.conf
chmod 644 .nginx/www.onesimus25.ru.conf

# 6. Перезапустите приложение
touch tmp/restart.txt
```

### Шаг 3: Проверьте

```bash
# Подождите 1-2 минуты, затем проверьте
curl -I https://onesimus25.ru/static/css/style.css

# Должно быть:
# HTTP/2 200
# Content-Type: text/css
```

## Если все еще 404

### Проверьте путь

```bash
# Убедитесь, что путь правильный
ls -la /home/o/onesim8n/onesimus/onesimus/staticfiles/css/style.css

# Если файл не найден, узнайте правильный путь:
pwd
realpath staticfiles/css/style.css
```

Используйте этот путь в конфигурации nginx.

### Проверьте логи Django

```bash
# Если nginx не работает, Django view должен обработать запрос
tail -f logs/django.log | grep -i "static"
```

Если видите сообщения `=== STATIC FILE REQUEST ===` - Django view вызывается, но файлы не находит.

### Проверьте DEBUG на сервере

```bash
# Проверьте текущее значение DEBUG
grep "^DEBUG" config/settings.py

# Если DEBUG = True, Django использует другой механизм
# Если DEBUG = False, Django использует кастомный view
```

## После исправления

1. Очистите кеш браузера (Ctrl+Shift+R)
2. Откройте консоль разработчика (F12) → Network
3. Обновите страницу
4. Проверьте запросы к CSS/JS - должен быть `200 OK` и правильный `Content-Type`
