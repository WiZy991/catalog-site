## Улучшенная логика определения категорий

### Приоритет ключевых слов
1. **Сначала проверяются категории из БД** (если есть keywords)
2. **Затем проверяются hardcoded категории**
3. **Ключевые слова сортируются по длине** (более длинные/специфичные первыми)
4. **Глобальная сортировка** - все ключевые слова из всех категорий сортируются вместе

### Примеры работы:
- "Датчик температуры двигателя" → найдет "датчик температуры" (двигатель) раньше общего "датчик"
- "Фильтр масляный" → найдет "фильтр масляный" (двигатель) раньше общего "фильтр"
- "Тормозная колодка" → найдет "тормозная колодка" раньше общего "тормоз"
