# Настройка автоматического импорта через планировщик Beget

## 📋 Пошаговая инструкция

### Шаг 1: Откройте планировщик заданий

1. Войдите в панель управления Beget
2. Найдите раздел **"Планировщик заданий"** или **"Crontab"**
3. Откройте его

### Шаг 2: Используйте "Мастер заданий"

1. Убедитесь, что выбрана вкладка **"Мастер заданий"** (Task Wizard)
2. Если нет - переключитесь на неё

### Шаг 3: Заполните форму

#### 3.1. Команда для выполнения

В поле **"Команда для выполнения"** вставьте:

```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --recent 10 >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

**Важно:** Замените пути на ваши реальные пути!

**Как узнать правильные пути:**
- Базовый путь: обычно `/home/o/onesim8n/onesimus/onesimus` (проверьте через SSH: `pwd`)
- Python: обычно `/home/o/onesim8n/onesimus/venv/bin/python` (проверьте через SSH: `which python`)

#### 3.2. Описание задания

В поле **"Описание задания"** введите:

```
Автоматический импорт товаров от 1С
```

или

```
Обработка XML файлов от 1С каждые 5 минут
```

#### 3.3. Настройка расписания

Выберите частоту выполнения:

**Вариант A: Каждые 5 минут** (рекомендуется для частых обменов)

1. В поле **"В указанные минуты"** введите: `*/5`
   - Это означает: каждые 5 минут
2. В поле **"В указанные часы"** оставьте пустым или введите: `*`
   - Это означает: каждый час
3. Нажмите кнопку **"В любой день недели"**
4. Нажмите кнопку **"Каждый день"**
5. Нажмите кнопку **"Каждый месяц"**

**Вариант B: Каждые 15 минут**

1. В поле **"В указанные минуты"** введите: `*/15`
2. В поле **"В указанные часы"** оставьте пустым или введите: `*`
3. Нажмите кнопку **"В любой день недели"**
4. Нажмите кнопку **"Каждый день"**
5. Нажмите кнопку **"Каждый месяц"**

**Вариант C: Каждый час**

1. В поле **"В указанные минуты"** введите: `0`
   - Это означает: в начале каждого часа (00 минут)
2. В поле **"В указанные часы"** оставьте пустым или введите: `*`
3. Нажмите кнопку **"В любой день недели"**
4. Нажмите кнопку **"Каждый день"**
5. Нажмите кнопку **"Каждый месяц"**

**Вариант D: Раз в день (в 3:00 ночи)**

1. В поле **"В указанные минуты"** введите: `0`
2. В поле **"В указанные часы"** введите: `3`
3. Нажмите кнопку **"В любой день недели"**
4. Нажмите кнопку **"Каждый день"**
5. Нажмите кнопку **"Каждый месяц"**

### Шаг 4: Сохраните задание

1. Нажмите кнопку **"Добавить задание"** (Add task)
2. Задание будет добавлено в список

### Шаг 5: Проверьте задание

1. После добавления задание должно появиться в списке заданий
2. Убедитесь, что оно активно (включено)

## 🔍 Альтернативный способ: "Составить задание вручную"

Если "Мастер заданий" не подходит, используйте вкладку **"Составить задание вручную"**:

1. Переключитесь на вкладку **"Составить задание вручную"**
2. В поле для cron выражения введите:

**Для каждых 5 минут:**
```
*/5 * * * *
```

**Для каждых 15 минут:**
```
*/15 * * * *
```

**Для каждого часа:**
```
0 * * * *
```

**Для раз в день (3:00):**
```
0 3 * * *
```

3. В поле команды вставьте:
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --recent 10 >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

4. Сохраните задание

## 📊 Проверка работы

### 1. Проверьте логи

Через SSH:
```bash
tail -f logs/cron_1c_import.log
```

Или через файловый менеджер Beget:
- Откройте файл `logs/cron_1c_import.log`

### 2. Проверьте обработку файлов

```bash
# Список обработанных файлов
ls -la media/1c_exchange/*.processed
```

### 3. Проверьте логи Django

```bash
tail -f logs/django.log | grep -i "обработк\|товар"
```

## ⚙️ Настройка email уведомлений

В интерфейсе Beget есть возможность настроить отправку результатов на email:

1. Найдите ссылку **"указанный email"** в левой панели
2. Нажмите на неё
3. Укажите ваш email
4. Результаты выполнения задач будут отправляться на этот email

## 🔧 Параметры команды

### Основные параметры:

- `--recent 10` - обработать только файлы, измененные за последние 10 минут
  - Рекомендуется для частых проверок (каждые 5 минут)
  - Для проверки каждый час используйте `--recent 65`
  - Для проверки раз в день можно убрать этот параметр

- `--zip` - обрабатывать также ZIP архивы (если 1С выгружает в ZIP)

- `--all` - обработать все файлы, даже если они уже обработаны

### Примеры команд:

**Для проверки каждые 5 минут:**
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --recent 10 >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

**Для проверки каждый час:**
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --recent 65 >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

**Для проверки раз в день:**
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

## ⚠️ Важно

1. **Пути:** Убедитесь, что пути в команде соответствуют вашей структуре проекта
   - Проверьте через SSH: `pwd` и `which python`

2. **Директория logs:** Убедитесь, что директория `logs/` существует:
   ```bash
   mkdir -p logs
   chmod 755 logs
   ```

3. **Права на файлы:** Убедитесь, что директория `media/1c_exchange/` доступна для записи:
   ```bash
   chmod 755 media/1c_exchange/
   ```

## ✅ Итог

После настройки:

1. ✅ 1С выгружает файл в `media/1c_exchange/`
2. ✅ Cron автоматически находит и обрабатывает файл
3. ✅ Товары автоматически создаются/обновляются на сайте
