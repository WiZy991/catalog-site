# Решение проблемы: товары не импортируются

## 🔴 Проблема

Скрипт выполняется успешно, но товары не появляются на сайте.

## 🔍 Причина

Файлы пропускаются из-за флага `--recent 10`, который обрабатывает только файлы, измененные за последние 10 минут. Если файлы старые (изменены ранее), они пропускаются.

В логах видно:
```
Пропускаем старый файл: import0_1.xml (изменен 2026-02-17 17:55:44)
Пропускаем старый файл: offers0_1.xml (изменен 2026-02-17 17:56:08)
```

## ✅ Решения

### Вариант 1: Запустить без ограничения по времени (рекомендуется)

Запустите скрипт без флага `--recent`:

```bash
cd /home/o/onesim8n/onesimus/onesimus
source venv/bin/activate
python manage.py process_1c_files
```

Это обработает все новые файлы (без маркера `.processed`).

### Вариант 2: Запустить с флагом `--all`

Обработает все файлы, даже если они уже обработаны:

```bash
python manage.py process_1c_files --all
```

### Вариант 3: Удалить маркеры и запустить снова

Удалите маркеры обработанных файлов и запустите скрипт:

```bash
cd /home/o/onesim8n/onesimus/onesimus/media/1c_exchange
rm -f *.processed
cd /home/o/onesim8n/onesimus/onesimus
source venv/bin/activate
python manage.py process_1c_files
```

### Вариант 4: Обработать конкретный файл

Если нужно обработать конкретный файл:

```bash
python manage.py process_1c_files --file import0_1.xml
```

## 🔧 Обновление cron-задачи

Если вы хотите, чтобы cron обрабатывал все новые файлы (не только за последние 10 минут), измените команду в cron:

### Текущая команда (только файлы за последние 10 минут):
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --recent 10 >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

### Рекомендуемая команда (все новые файлы):
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

**Разница:** убрали `--recent 10` - теперь будут обрабатываться все файлы без маркера `.processed`, независимо от времени изменения.

## 📊 Проверка результата

После запуска скрипта проверьте:

1. **Вывод скрипта:**
   ```bash
   tail -20 logs/cron_1c_import.log
   ```

   Должно быть что-то вроде:
   ```
   ✓ import0_1.xml: обработано 150 товаров (создано: 50, обновлено: 100)
   ```

2. **Проверьте товары в админке:**
   - Зайдите в админку Django
   - Откройте раздел "Товары"
   - Проверьте, что товары появились

3. **Проверьте товары на сайте:**
   - Откройте каталог на сайте
   - Товары должны отображаться

## 🎯 Рекомендации

1. **Для первого запуска:** используйте `python manage.py process_1c_files` (без флагов)
2. **Для cron:** используйте команду без `--recent 10`, чтобы обрабатывались все новые файлы
3. **Для повторной обработки:** используйте `--all` или удалите маркеры `.processed`

## ⚠️ Важно

- Файлы с маркером `.processed` не обрабатываются повторно (защита от дублирования)
- Если файл был изменен после обработки, он будет обработан снова
- Товары создаются с `is_active=True` и `catalog_type='retail'`, поэтому они должны отображаться на сайте
