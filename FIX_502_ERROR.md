# Исправление 502 Bad Gateway

## Проблема
Синтаксическая ошибка в `catalog/services.py` (строка 372 - неправильный отступ) вызывала 502 Bad Gateway.

## Решение

### Вариант 1: Через FileZilla
1. Загрузите исправленный файл `catalog/services.py` на сервер
2. Перезапустите приложение через панель управления хостингом

### Вариант 2: Через SSH (если есть доступ)

```bash
# 1. Перейдите в директорию проекта
cd ~/onesimus/onesimus
# или
cd /path/to/your/project

# 2. Проверьте синтаксис файла
python -m py_compile catalog/services.py

# 3. Если ошибок нет, перезапустите приложение
# Для Beget обычно:
touch tmp/restart.txt

# Или через systemd (если используется):
sudo systemctl restart gunicorn
# или
sudo systemctl restart uwsgi

# 4. Проверьте логи на наличие ошибок
tail -f /path/to/logs/error.log
# или
journalctl -u gunicorn -f
```

### Вариант 3: Через панель управления Beget
1. Зайдите в панель управления Beget
2. Найдите раздел "Python приложения" или "Веб-приложения"
3. Найдите ваше приложение и нажмите "Перезапустить"

## Проверка
После перезапуска откройте сайт в браузере. Если ошибка 502 исчезла, значит проблема решена.

## Примечание
Ошибка `safari is not defined` в консоли браузера - это проблема расширения браузера (gosuslugi.plugin.extension), не связана с вашим сайтом. Её можно игнорировать.
