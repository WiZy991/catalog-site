# Проверка HTTP ответов для статики

## Выполните на сервере:

```bash
# Проверьте заголовки для style.css (который не работает)
curl -I https://onesimus25.ru/static/css/style.css

# Проверьте заголовки для cart.css (который работает)
curl -I https://onesimus25.ru/static/css/cart.css

# Проверьте полный ответ для style.css
curl -v https://onesimus25.ru/static/css/style.css 2>&1 | tail -20

# Проверьте полный ответ для cart.css
curl -v https://onesimus25.ru/static/css/cart.css 2>&1 | tail -20

# Проверьте main.js
curl -I https://onesimus25.ru/static/js/main.js
```

## Что смотреть:

1. **HTTP статус**: должен быть `200 OK` для обоих
2. **Content-Type**: 
   - Для CSS должен быть `text/css` или `text/css; charset=utf-8`
   - Для JS должен быть `application/javascript`
3. **Content-Length**: должен совпадать с размером файла
4. **Размер ответа**: сравните с размером файла

## Если видите различия:

- Если `style.css` возвращает `404` или `text/html` - запрос не доходит до Django view
- Если `cart.css` возвращает `200` и `text/css` - view работает для этого файла
- Если оба возвращают одинаково, но один не работает - проблема в браузере/кеше
