# Перезапуск приложения после обновления кода

## Проблема

Код обновлён, но ошибка всё ещё возникает, потому что Python использует закешированную версию (.pyc файлы) или приложение не перезапущено.

## Решение

### 1. Очистите кеш Python

```bash
cd /home/o/onesim8n/onesimus/onesimus
find . -type d -name __pycache__ -exec rm -r {} + 2>/dev/null || true
find . -name "*.pyc" -delete
```

### 2. Перезапустите приложение

```bash
touch tmp/restart.txt
```

Или, если используется Passenger:

```bash
# Создайте или обновите файл restart.txt
touch tmp/restart.txt

# Или перезапустите через Passenger
passenger-config restart-app /home/o/onesim8n/onesimus/onesimus
```

### 3. Проверьте, что файл действительно обновлён

```bash
# Проверьте дату изменения файла
ls -la catalog/commerceml_views.py

# Проверьте, что try-except есть в коде
grep -n "try:" catalog/commerceml_views.py | grep -A 2 "ProductCharacteristic"
```

Должна быть строка с `try:` перед `ProductCharacteristic.objects.filter(product=product).delete()`

### 4. Если всё ещё не работает

Попробуйте полный перезапуск:

```bash
# Остановите приложение (если запущено вручную)
# Или просто обновите restart.txt
touch tmp/restart.txt

# Подождите несколько секунд
sleep 5

# Проверьте логи, чтобы убедиться, что приложение перезапустилось
tail -f logs/django.log
```

### 5. Альтернативный способ - перезапуск через веб-интерфейс

Если у вас есть доступ к панели управления хостингом (Beget), можно перезапустить приложение через неё.

## Проверка после перезапуска

После перезапуска попробуйте снова импортировать товары. Ошибка должна исчезнуть, и код должен обрабатывать отсутствие таблицы ProductCharacteristic корректно (с предупреждением в логах, но без падения).
