# Диагностика проблемы с cron

## ✅ Что работает

- Ручной запуск: `python manage.py process_1c_files` - **работает**
- Товары добавляются при ручном запуске

## ❌ Что не работает

- Автоматический запуск через cron - **не работает**

## 🔍 Диагностика

### Проблема 1: В cron используется `--recent 10`

Из логов видно:
```
Обрабатываем файлы, измененные за последние 10 минут
Пропускаем старый файл: import0_1.xml (изменен 2026-02-18 15:04:08)
```

Это значит, что в cron-задаче все еще используется флаг `--recent 10`.

### Проблема 2: Файлы уже помечены как обработанные

Из логов видно:
```
Пропускаем уже обработанный файл: import0_1.xml
Пропускаем уже обработанный файл: offers0_1.xml
```

## ✅ Решение

### Шаг 1: Проверьте текущую команду в cron

1. Зайдите в Beget → "Планировщик заданий"
2. Найдите задание для импорта
3. Посмотрите команду

**Неправильная команда (с `--recent 10`):**
```bash
... process_1c_files --recent 10 >> ...
```

**Правильная команда (без `--recent 10`):**
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

### Шаг 2: Измените команду в cron

1. Откройте задание в Beget
2. Измените команду - **уберите `--recent 10`**
3. Сохраните изменения

### Шаг 3: Удалите маркеры для повторной обработки

Через SSH:

```bash
cd /home/o/onesim8n/onesimus/onesimus/media/1c_exchange
rm -f *.processed
```

Это позволит cron обработать файлы снова.

### Шаг 4: Проверьте работу cron

Подождите 5-10 минут и проверьте лог:

```bash
tail -f logs/cron_1c_import.log
```

Или через файловый менеджер Beget откройте `logs/cron_1c_import.log`.

## 🔧 Альтернативное решение: Использовать `--all` для теста

Если хотите протестировать cron с обработкой всех файлов:

```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --all >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

Но для постоянной работы лучше без `--all` и без `--recent` - тогда будут обрабатываться только новые файлы.

## 📊 Проверка правильности команды

### Правильная команда для cron:

```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

**Ключевые моменты:**
- ✅ Абсолютные пути
- ✅ Нет флага `--recent 10`
- ✅ Нет флага `--all` (для постоянной работы)
- ✅ Логирование в файл

```

```

(каждые 5 минут)

## ⚠️ Частые ошибки

### Ошибка 1: Флаг `--recent 10` в команде

**Признак:** В логах видно "Обрабатываем файлы, измененные за последние 10 минут"

**Решение:** Убрать `--recent 10` из команды

### Ошибка 2: Файлы уже обработаны

**Признак:** В логах видно "Пропускаем уже обработанный файл"

**Решение:** Удалить маркеры `.processed` или использовать `--all` для теста

### Ошибка 3: Неправильные пути

**Признак:** В логах ошибки "No such file or directory"

**Решение:** Использовать абсолютные пути в команде
