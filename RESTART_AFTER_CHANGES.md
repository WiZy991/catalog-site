# Что делать после изменений через файловый менеджер Beget

## Быстрая инструкция

### После изменений в Python коде (services.py, views.py, models.py и т.д.):

**Способ 1: Через файловый менеджер Beget**
1. Создайте папку `tmp` в корне проекта (если её нет)
2. Создайте или обновите файл `tmp/restart.txt` (можно просто создать пустой файл или добавить туда текущую дату/время)
3. Сохраните файл - приложение перезапустится автоматически

**Способ 2: Через панель управления Beget**
1. Войдите в панель управления Beget
2. Перейдите в раздел "Сайты" → выберите ваш сайт
3. Найдите кнопку "Перезапустить приложение" или "Restart"
4. Нажмите её

**Способ 3: Через SSH (если есть доступ)**
```bash
cd ~/onesimus/onesimus
touch tmp/restart.txt
```

### После изменений в статических файлах (CSS, JS):

**Через SSH или терминал в панели Beget:**
```bash
cd ~/onesimus/onesimus
source venv/bin/activate  # если виртуальное окружение используется
python manage.py collectstatic --noinput
touch tmp/restart.txt
```

### После изменений в шаблонах (HTML):

Достаточно перезапустить приложение (см. выше)

### После изменений в настройках (settings.py):

**ОБЯЗАТЕЛЬНО перезапустить приложение!**

---

## Важные моменты:

1. ✅ **Python код** → всегда перезапуск
2. ✅ **Настройки (settings.py)** → всегда перезапуск
3. ✅ **Шаблоны (HTML)** → перезапуск (для надежности)
4. ✅ **Статические файлы (CSS/JS)** → `collectstatic` + перезапуск
5. ✅ **Медиа файлы (изображения)** → обычно ничего не нужно

## Проверка что изменения применились:

1. Очистите кеш браузера (Ctrl+Shift+R или Ctrl+F5)
2. Откройте сайт в режиме инкогнито
3. Проверьте логи ошибок в панели Beget (если есть ошибки)

## Если изменения не применяются:

1. Убедитесь, что файл `tmp/restart.txt` был создан/обновлен
2. Подождите 10-30 секунд после перезапуска
3. Проверьте логи приложения в панели Beget
4. Для статических файлов убедитесь, что выполнили `collectstatic`
