# Настройка автоматического импорта товаров от 1С

## 🎯 Цель

Настроить автоматическую обработку XML файлов от 1С без необходимости вручную скачивать файлы и импортировать их через админку.

## 📋 Как это работает

1. **1С выгружает файл** → файл сохраняется в `media/1c_exchange/`
2. **Cron задача проверяет папку** → находит новые XML файлы
3. **Система автоматически обрабатывает** → создает/обновляет товары
4. **Готово!** → товары на сайте обновлены автоматически

## 🔧 Параметры команды

### Основные параметры:

- `--recent N` - обработать только файлы, измененные за последние N минут
  - Пример: `--recent 10` - обработает файлы, измененные за последние 10 минут
  - Полезно для частых проверок, чтобы не обрабатывать старые файлы

- `--all` - обработать все файлы, даже если они уже обработаны
  - Используйте для повторной обработки всех файлов

- `--file filename.xml` - обработать конкретный файл
  - Пример: `--file import.xml`

- `--zip` - обрабатывать также ZIP архивы
  - По умолчанию обрабатываются только XML файлы

### Примеры использования:

```bash
# Обработать все новые XML файлы
python manage.py process_1c_files

# Обработать только файлы, измененные за последние 10 минут
python manage.py process_1c_files --recent 10

# Обработать все файлы, включая ZIP
python manage.py process_1c_files --zip

# Обработать конкретный файл
python manage.py process_1c_files --file import.xml

# Обработать все файлы заново
python manage.py process_1c_files --all
```

### Проверка обработанных файлов:

```bash
# Список обработанных файлов (с маркерами .processed)
ls -la media/1c_exchange/*.processed

# Список всех файлов
ls -la media/1c_exchange/
```

## 🔍 Как определить, что файл обработан

После обработки файла создается маркерный файл с расширением `.processed`:
- Файл: `import.xml`
- Маркер: `import.xml.processed`

Если маркер существует, файл считается обработанным и будет пропущен при следующем запуске (если не используется `--all`).

## ⚠️ Важно

1. **Пути:** Убедитесь, что пути в cron задаче соответствуют вашей структуре проекта:
   - Базовый путь: `/home/o/onesim8n/onesimus/onesimus`
   - Python: `/home/o/onesim8n/onesimus/venv/bin/python`
   - Проверьте свои пути командой: `pwd` и `which python`

2. **Права на файлы:** Убедитесь, что директория доступна для записи:
   ```bash
   chmod 755 media/1c_exchange/
   ```

3. **Логи:** Логи сохраняются в `logs/cron_1c_import.log`. Убедитесь, что директория `logs/` существует:
   ```bash
   mkdir -p logs
   chmod 755 logs
   ```

4. **Частота проверки:** Выберите частоту проверки в зависимости от того, как часто 1С выгружает файлы:
   - Если обмен раз в день → проверка раз в день
   - Если обмен несколько раз в день → проверка каждые 15-30 минут
   - Если обмен очень частый → проверка каждые 5 минут

## 🔄 Процесс работы

```
1С → [Выполнить обмен] → Файл в media/1c_exchange/
                                    ↓
                            Cron проверяет папку
                                    ↓
                            Находит новый файл
                                    ↓
                            Обрабатывает XML
                                    ↓
                    Создает/обновляет товары на сайте
                                    ↓
                            Готово! ✅
```

## 📝 Примечания

- Файлы не удаляются автоматически после обработки (для безопасности)
- Можно настроить автоматическое удаление старых файлов через отдельную cron задачу
- Все действия логируются для отладки
