# Инструкция по настройке Cron для обмена с 1С

## ⚠️ ВАЖНО: Что должен делать Cron

Cron должен **ТОЛЬКО** проверять файлы от 1С на сервере и обновлять товары при наличии изменений.

**Cron НЕ должен:**
- Деактивировать товары
- Менять статус наличия товаров
- Запускать команды активации/деактивации

## ✅ Правильная команда для Cron

В планировщике заданий (Beget) должна быть **ТОЛЬКО** одна команда:

```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

**Cron выражение:** `*/5 * * * *` (каждые 5 минут)

## ❌ Команды, которые НЕ должны быть в Cron

**НЕ добавляйте в cron следующие команды:**
- `activate_products` - эта команда деактивирует товары без остатка и цены
- `activate_wholesale_products` - эта команда деактивирует товары без остатка

Эти команды нужны только для ручной проверки и исправления состояния товаров, но **НЕ должны запускаться автоматически**.

## 🔧 Как работает process_1c_files

Команда `process_1c_files`:
1. Проверяет директорию `media/1c_exchange/` на наличие новых XML файлов
2. Обрабатывает только файлы без маркера `.processed` (новые файлы)
3. Обновляет товары из файлов от 1С
4. Автоматически активирует/деактивирует товары в зависимости от остатков и цен из файлов 1С
5. Создает маркер `.processed` для обработанных файлов

**Важно:** Активация/деактивация товаров происходит автоматически при обработке файлов от 1С, на основе данных из файлов. Не нужно запускать отдельные команды активации.

## 📋 Проверка текущих настроек Cron

1. Зайдите в панель управления Beget
2. Откройте "Планировщик заданий" (Cron)
3. Проверьте все активные задания

**Должно быть только одно задание:**
- Название: "Автоматический импорт товаров от 1С" (или похожее)
- Команда: `process_1c_files` (без `activate_products` или `activate_wholesale_products`)

**Если есть другие задания с командами активации - УДАЛИТЕ их!**

## 🛠️ Если нужно исправить состояние товаров вручную

Если после обмена с 1С товары не активировались правильно, можно запустить команды активации **ВРУЧНУЮ** через SSH:

```bash
cd /home/o/onesim8n/onesimus/onesimus
source venv/bin/activate
python manage.py activate_products
```

Но эти команды **НЕ должны быть в cron**!

## 📊 Логирование

Все действия cron записываются в файл:
```
/home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log
```

Проверяйте этот файл для диагностики проблем.

## ✅ Итоговая настройка

1. **В Cron должна быть ТОЛЬКО одна команда:**
   ```bash
   cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
   ```

2. **Расписание:** `*/5 * * * *` (каждые 5 минут)

3. **Удалите все другие задания**, которые запускают `activate_products` или `activate_wholesale_products`

## 🔍 Проверка работы

После настройки:
1. Подождите 5-10 минут
2. Проверьте файл `logs/cron_1c_import.log`
3. Убедитесь, что товары обновляются из файлов 1С
4. Убедитесь, что товары не деактивируются без причины

Если товары всё равно деактивируются - проверьте, что в cron нет команд активации!
