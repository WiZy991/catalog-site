13. Раздел «Оптовые продажи / Для партнёров»
13.1. Назначение раздела
На сайте должен быть реализован отдельный раздел «Оптовые продажи» (альтернативное название — «Для партнёров»), предназначенный для работы с оптовыми клиентами и постоянными партнёрами.
Цель раздела:
привлечение новых оптовых клиентов;
сбор заявок на партнёрство;
предоставление зарегистрированным партнёрам доступа к оптовым ценам и данным о наличии товаров.

13.2. Структура раздела
Раздел «Для партнёров» должен включать:
Информационный блок:
описание условий сотрудничества;
преимущества работы с компанией;
порядок получения оптового доступа.
Кнопку «Регистрация партнёра».
Каталог товаров для партнёров (ограниченный режим до регистрации).

13.3. Регистрация партнёра
13.3.1. Форма регистрации
При нажатии кнопки «Регистрация партнёра» пользователь заполняет форму заявки, содержащую (минимальный набор):
ФИО контактного лица;
телефон;
e-mail;
город;
комментарий (Не обязательно).
13.3.2. Логика обработки заявки
После отправки формы:
заявка отправляется ответственному менеджеру по оптовым продажам / партнёрству;
указать конкретный e-mail менеджера по опту;
заявка сохраняется в административной панели;
Пользователю отображается сообщение:
«Ваша заявка принята. С вами свяжется менеджер для подтверждения доступа».

13.4. Активация партнёра и авторизация
После ручного подтверждения заявки администратором:
создаётся учётная запись партнёра;
пользователю предоставляются данные для авторизации (или активируется доступ).
Авторизация партнёров осуществляется через:
логин + пароль.
13.5. Поведение каталога в разделе «Для партнёров»
13.5.1. До регистрации / без авторизации
Пользователь видит:
структуру каталога;
карточки товаров;
изображения;
названия товаров;
основные характеристики.
При этом НЕ отображаются:
цены;
наличие;
оптовые условия;
данные, чувствительные для бизнеса.
Вместо цены отображается:
текст «Цена доступна после регистрации»
или
кнопка «Получить оптовый доступ».

13.5.2. После регистрации и авторизации
Зарегистрированный партнёр видит:
оптовые цены;
актуальное наличие товаров;
полную информацию по позициям;
тот же каталог, но в расширенном режиме доступа.
13.6. Управление партнёрами в CMS
В административной панели должно быть реализовано:
просмотр заявок от партнёров;
подтверждение / отклонение заявок;
редактирование данных партнёров;
включение / отключение оптового доступа;
возможность назначения:
оптовых цен,
групп цен (при необходимости).

13.7. Оптовые цены
Для каждого товара должна быть возможность:
задавать розничную цену;
задавать оптовую цену.
Оптовая цена отображается только авторизованным партнёрам.
Логика отображения цен должна быть гибкой и масштабируемой.

13.8. Безопасность и ограничения доступа
Данные о ценах и наличии должны быть недоступны неавторизованным пользователям.
Прямой доступ по URL к оптовым данным должен быть защищён.
Индексация раздела «Для партнёров» поисковыми системами должна быть отключена.


13.9  Импорт товаров через CMS
В административной панели сайта должен быть реализован механизм массового импорта товаров в каталог.
13.9.1. Форматы импорта
Поддерживаемые форматы файлов:
XLS
XLSX
CSV
