# Решение проблемы "database is locked"

## 🔴 Проблема

При попытке открыть карточку товара возникает ошибка:
```
OperationalError: database is locked
```

## 🔍 Причина

SQLite не поддерживает параллельные записи. Когда скрипт импорта работает и выполняет долгие транзакции, база данных блокируется, и веб-приложение не может получить доступ.

## ✅ Решение

### 1. Добавлены настройки таймаутов для SQLite

В файле `config/settings.py` добавлены параметры:
- `timeout: 30` - время ожидания разблокировки базы (30 секунд)
- `check_same_thread: False` - разрешить использование из разных потоков
- `CONN_MAX_AGE: 0` - не использовать постоянные соединения

### 2. Что нужно сделать на сервере

1. **Загрузите обновленный файл `config/settings.py`** на сервер

2. **Перезапустите приложение:**
   - Если используется Passenger, изменения применятся автоматически
   - Или перезапустите сервер вручную

3. **Проверьте, что скрипт импорта не работает слишком долго:**
   ```bash
   ps aux | grep process_1c_files
   ```
   
   Если скрипт работает, дождитесь его завершения.

### 3. Оптимизация импорта

Транзакции уже оптимизированы - каждый товар обрабатывается в отдельной транзакции, что уменьшает время блокировки.

## 🔧 Дополнительные рекомендации

### Если проблема сохраняется:

1. **Проверьте, не запущено ли несколько процессов импорта:**
   ```bash
   ps aux | grep python | grep process_1c_files
   ```

2. **Убейте зависшие процессы (если есть):**
   ```bash
   pkill -f process_1c_files
   ```

3. **Проверьте блокировку базы данных:**
   ```bash
   cd /home/o/onesim8n/onesimus/onesimus
   sqlite3 db.sqlite3 "PRAGMA busy_timeout;"
   ```

4. **Принудительно разблокируйте базу (если нужно):**
   ```bash
   # Закройте все соединения с базой
   # Перезапустите веб-сервер
   ```

### Долгосрочное решение

Для production рекомендуется использовать PostgreSQL или MySQL вместо SQLite, так как они:
- Поддерживают параллельные записи
- Лучше работают под нагрузкой
- Имеют лучшую производительность

Но для текущей задачи SQLite с таймаутами должен работать.

## 📊 Проверка

После применения изменений:

1. **Попробуйте открыть карточку товара** - ошибка должна исчезнуть
2. **Если ошибка все еще возникает:**
   - Подождите 30 секунд и попробуйте снова (таймаут)
   - Проверьте, не работает ли скрипт импорта
   - Перезапустите веб-сервер

## ⚠️ Важно

- Таймаут 30 секунд означает, что веб-приложение будет ждать до 30 секунд, пока база разблокируется
- Если импорт работает очень долго, лучше запускать его в нерабочее время
- Для больших импортов рекомендуется использовать фоновые задачи (Celery) или обрабатывать файлы ночью
