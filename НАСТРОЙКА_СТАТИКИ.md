# Настройка раздачи статических файлов

## Что было сделано

Настроена правильная раздача статических файлов для работы как при `DEBUG=True`, так и при `DEBUG=False`.

### Изменения в `config/urls.py`:

1. **При DEBUG=True** (разработка):
   - Используется стандартный механизм Django `staticfiles_urlpatterns()`
   - Автоматически обрабатывает файлы из `STATICFILES_DIRS` и `STATIC_ROOT`
   - Это стандартный и рекомендуемый способ для разработки

2. **При DEBUG=False** (продакшен):
   - Используется кастомный view `serve_static_file`
   - Работает как fallback, если nginx не настроен
   - Ищет файлы сначала в `STATIC_ROOT`, затем в `STATICFILES_DIRS`

## Текущие настройки в `settings.py`:

```python
STATIC_URL = '/static/'
STATICFILES_DIRS = [BASE_DIR / 'static']  # Исходные файлы
STATIC_ROOT = BASE_DIR / 'staticfiles'    # Собранные файлы для продакшена
```

## Что нужно сделать на сервере

### 1. Убедитесь, что статические файлы собраны:

```bash
cd ~/onesimus/onesimus
python manage.py collectstatic --noinput
```

### 2. Проверьте права доступа:

```bash
chmod -R 755 staticfiles/
chmod -R 644 staticfiles/**/*
```

### 3. Настройте nginx (рекомендуется для продакшена):

Создайте или обновите файл `.nginx/onesimus25.ru.conf`:

```nginx
location /static/ {
    alias /home/o/onesim8n/onesimus/onesimus/staticfiles/;
    expires 30d;
    add_header Cache-Control "public, immutable";
    access_log off;
}

location /media/ {
    alias /home/o/onesim8n/onesimus/onesimus/media/;
    expires 7d;
    add_header Cache-Control "public";
    access_log off;
}
```

Также создайте такой же файл для `www.onesimus25.ru.conf`.

### 4. Перезапустите приложение:

```bash
touch tmp/restart.txt
```

### 5. Проверьте работу статики:

```bash
# Проверьте доступность CSS
curl -I https://onesimus25.ru/static/css/style.css

# Проверьте доступность JS
curl -I https://onesimus25.ru/static/js/main.js

# Проверьте доступность изображений
curl -I https://onesimus25.ru/static/images/logo.png
```

Должен вернуться `HTTP/2 200` или `HTTP/1.1 200 OK`.

## Важные замечания

1. **При DEBUG=True**: Django автоматически раздает статику через встроенный механизм. Это работает "из коробки".

2. **При DEBUG=False**: 
   - Предпочтительно использовать nginx для раздачи статики (быстрее и эффективнее)
   - Кастомный view работает как fallback, если nginx не настроен
   - Django view медленнее nginx, но работает в любом случае

3. **После изменений в CSS/JS**: Всегда выполняйте `collectstatic` на сервере!

## Диагностика проблем

Если статика не загружается:

1. Проверьте, что файлы существуют в `staticfiles/`:
   ```bash
   ls -la staticfiles/css/style.css
   ```

2. Проверьте логи Django:
   ```bash
   tail -f logs/django.log
   ```

3. Проверьте, что путь в nginx правильный (если используется):
   ```bash
   cat .nginx/onesimus25.ru.conf
   ```

4. Проверьте права доступа:
   ```bash
   ls -la staticfiles/
   ```

## Тестирование локально

Для тестирования с DEBUG=False локально:

1. Временно установите `DEBUG = False` в `settings.py`
2. Запустите сервер: `python manage.py runserver`
3. Проверьте доступность статики: `http://localhost:8000/static/css/style.css`

После тестирования верните `DEBUG = True` для разработки.
