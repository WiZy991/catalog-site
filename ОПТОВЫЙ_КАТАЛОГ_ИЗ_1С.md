# Загрузка оптовых товаров из 1С

## 📋 Описание

Система теперь поддерживает загрузку товаров из 1С как в розничный, так и в оптовый каталог. Товары автоматически распределяются по каталогам в зависимости от имени файла.

## 🔄 Как это работает

### Определение типа каталога

Тип каталога определяется **автоматически по имени файла**:

- **Розничный каталог (retail):**
  - `import.xml`
  - `import_retail.xml`
  - `offers.xml`
  - `offers_retail.xml`
  - Любые другие файлы без ключевых слов опта

- **Оптовый каталог (wholesale):**
  - `import_wholesale.xml`
  - `import_opt.xml`
  - `import_опт.xml`
  - `offers_wholesale.xml`
  - `offers_opt.xml`
  - `offers_опт.xml`
  - Любые файлы с ключевыми словами: `wholesale`, `опт`, `opt`

### Ключевые слова для опта

Система определяет оптовый каталог, если в имени файла есть одно из слов:
- `wholesale`
- `опт`
- `opt`
- `wholesale_` (с подчеркиванием)
- `_wholesale` (с подчеркиванием)

## 📁 Структура файлов от 1С

### Вариант 1: Разные файлы для розницы и опта

```
import_retail.xml      → Розничный каталог
import_wholesale.xml   → Оптовый каталог
offers_retail.xml      → Цены и остатки для розницы
offers_wholesale.xml   → Цены и остатки для опта
```

### Вариант 2: Один файл для каждого типа

```
import.xml             → Розничный каталог (по умолчанию)
import_wholesale.xml   → Оптовый каталог
offers.xml             → Цены и остатки для розницы
offers_wholesale.xml   → Цены и остатки для опта
```

## 🔧 Что обрабатывается

### Для оптового каталога работает всё то же самое:

1. **Импорт товаров** (`import_wholesale.xml`):
   - Создание/обновление товаров
   - Определение категорий
   - Парсинг характеристик
   - Автоматическое определение бренда и артикула

2. **Обновление цен и остатков** (`offers_wholesale.xml`):
   - Обновление цен (`price` и `wholesale_price`)
   - Обновление остатков (`quantity`)
   - Автоматическое скрытие товаров без остатка

3. **Синхронизация остатков**:
   - Товары без остатка (`quantity = 0`) автоматически скрываются
   - Товары с остатком (`quantity > 0`) отображаются

4. **Удаление товаров**:
   - Товары, которые не пришли в обмене, автоматически скрываются
   - Работает отдельно для розницы и опта

## 📊 Логика работы

```
1С → Файлы → Сайт
    ↓         ↓
import_retail.xml → Розничный каталог (retail)
import_wholesale.xml → Оптовый каталог (wholesale)

offers_retail.xml → Цены/остатки для розницы
offers_wholesale.xml → Цены/остатки для опта
```

## ✅ Что нужно сделать в 1С

1. **Настроить выгрузку оптовых товаров:**
   - Создайте отдельный обмен для оптового каталога
   - Назовите файлы с ключевым словом `wholesale`, `опт` или `opt`
   - Например: `import_wholesale.xml`, `offers_wholesale.xml`

2. **Настроить URL в 1С:**
   - Для розницы: `http://onesimus.ru/cml/exchange?type=catalog&mode=checkauth`
   - Для опта: тот же URL (система определит тип по имени файла)

3. **Загрузить файлы:**
   - Загрузите `import_wholesale.xml` - товары попадут в оптовый каталог
   - Загрузите `offers_wholesale.xml` - обновятся цены и остатки для опта

## 🔍 Проверка работы

### После загрузки файлов проверьте:

1. **Розничный каталог:**
   - Товары должны быть видны на основном сайте
   - URL: `http://onesimus.ru/catalog/`

2. **Оптовый каталог:**
   - Товары должны быть видны в разделе для партнёров
   - URL: `http://onesimus.ru/partners/catalog/`
   - Требуется авторизация партнёра

3. **Логи:**
   - Проверьте логи на наличие сообщений о типе каталога
   - Должны быть сообщения: "Определен тип каталога: ОПТОВЫЙ (wholesale)"

## ⚠️ Важные моменты

1. **Раздельные каталоги:**
   - Розничные и оптовые товары хранятся отдельно
   - Один и тот же товар может быть и в рознице, и в опте (с разными `external_id`)

2. **Синхронизация остатков:**
   - Работает отдельно для каждого типа каталога
   - Товары без остатка скрываются автоматически

3. **Удаление товаров:**
   - Работает отдельно для каждого типа каталога
   - Если товар удален в 1С из опта, он скрывается только в оптовом каталоге

4. **Цены:**
   - Розничные товары используют поле `price`
   - Оптовые товары используют поле `wholesale_price` (если указано) или `price`
