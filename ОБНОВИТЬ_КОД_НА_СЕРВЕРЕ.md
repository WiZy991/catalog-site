# Обновление кода на сервере

## Проблема

Ошибка всё ещё возникает, потому что на сервере старая версия кода без исправлений обработки ошибок.

## Решение

### 1. Обновите код на сервере

```bash
cd /home/o/onesim8n/onesimus/onesimus
source venv/bin/activate
git pull origin main
```

Это загрузит исправленные версии файлов:
- `catalog/commerceml_views.py` (с обработкой ошибок для ProductCharacteristic)
- `catalog/one_c_views.py` (с обработкой ошибок для ProductCharacteristic)

### 2. Перезапустите приложение

```bash
touch tmp/restart.txt
```

### 3. Проверьте, что код обновился

Проверьте, что в файле `catalog/commerceml_views.py` на строке ~1367 есть try-except блок:

```bash
grep -A 5 "Обрабатываем характеристики для ProductCharacteristic" catalog/commerceml_views.py
```

Должны увидеть что-то вроде:
```python
# Обрабатываем характеристики для ProductCharacteristic (если нужно)
if product_data.get('characteristics') and isinstance(product_data.get('characteristics'), list):
    try:
        # Удаляем старые ProductCharacteristic
        ProductCharacteristic.objects.filter(product=product).delete()
```

### 4. Если код не обновился

Если после `git pull` код всё ещё старый, возможно изменения не были отправлены в репозиторий. На локальном компьютере выполните:

```bash
git add catalog/commerceml_views.py catalog/one_c_views.py
git commit -m "Исправлена обработка ошибок ProductCharacteristic"
git push origin main
```

Затем на сервере снова:
```bash
git pull origin main
touch tmp/restart.txt
```

## После обновления

После обновления кода импорт должен работать, даже если таблица `catalog_productcharacteristic` ещё не создана (миграции не применены). Код будет просто пропускать создание ProductCharacteristic, но товары будут обрабатываться нормально.
