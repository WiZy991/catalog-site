# Проверка и настройка автоматического импорта через cron

## ✅ Текущая ситуация

Товары успешно импортируются при ручном запуске:
```bash
python manage.py process_1c_files
```

Но автоматический запуск через cron не работает.

## 🔍 Проверка текущей настройки cron

### 1. Проверьте, настроена ли cron-задача в Beget

1. Зайдите в панель управления Beget
2. Откройте раздел **"Планировщик заданий"** (Cron)
3. Проверьте, есть ли там задание для импорта товаров

### 2. Если задание есть - проверьте команду

Команда должна быть:
```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

**Важно:** Без флага `--recent 10` (чтобы обрабатывались все новые файлы)

### 3. Проверьте расписание

Расписание должно быть: `*/5 * * * *` (каждые 5 минут)

## 🔧 Настройка cron в Beget

### Вариант 1: Через "Мастер заданий"

1. Откройте **"Планировщик заданий"** в Beget
2. Нажмите **"Мастер заданий"** или **"Добавить задание"**
3. Заполните форму:
   - **Команда для выполнения:**
     ```bash
     cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
     ```
   - **Описание:** `Автоматический импорт товаров от 1С`
   - **Расписание:**
     - В указанные минуты: `*/5` (каждые 5 минут)
     - В указанные часы: оставьте пустым или `*`
     - Нажмите: "В любой день недели", "Каждый день", "Каждый месяц"
4. Нажмите **"Добавить задание"**

### Вариант 2: Через "Составить задание вручную"

1. Откройте **"Планировщик заданий"** в Beget
2. Нажмите **"Составить задание вручную"**
3. Заполните:
   - **Cron выражение:** `*/5 * * * *`
   - **Команда:**
     ```bash
     cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
     ```
4. Сохраните задание

- Есть ли ошибки

## 🔍 Диагностика проблем

### Проблема 1: Cron не запускается

**Признаки:**
- Файл `logs/cron_1c_import.log` не обновляется
- В логах Beget нет записей о запуске

**Решение:**
1. Проверьте правильность команды (абсолютные пути)
2. Проверьте права на выполнение:
   ```bash
   ls -la /home/o/onesim8n/onesimus/venv/bin/python
   ```
3. Проверьте, что файл `manage.py` существует:
   ```bash
   ls -la /home/o/onesim8n/onesimus/onesimus/manage.py
   ```

### Проблема 2: Cron запускается, но файлы не обрабатываются

**Признаки:**
- В логе видно "Найдено файлов: X", но "Обработано файлов: 0"
- Файлы пропускаются

**Решение:**
1. Проверьте, что файлы новые (без маркера `.processed`)
2. Убедитесь, что команда без флага `--recent 10`
3. Попробуйте запустить с флагом `--all` для теста:
   ```bash
   python manage.py process_1c_files --all
   ```

### Проблема 3: Ошибки в логах

**Признаки:**
- В логе есть ошибки (ERROR, Exception, Traceback)

**Решение:**
1. Проверьте, применены ли миграции:
   ```bash
   python manage.py migrate
   ```
2. Проверьте права на директорию обмена:
   ```bash
   ls -la media/1c_exchange/
   ```
3. Проверьте, что директория существует:
   ```bash
   ls -la media/1c_exchange/
   ```

## 🎯 Ожидаемое поведение

После правильной настройки cron:

1. **Каждые 5 минут** cron запускает скрипт
2. Скрипт проверяет директорию `media/1c_exchange/` на наличие новых XML файлов

5. Товары автоматически добавляются/обновляются на сайте

## 🔄 Альтернативные варианты расписания

### Каждые 5 минут (рекомендуется):
- Cron: `*/5 * * * *`
- Подходит для частых обновлений от 1С

### Каждые 15 минут:
- Cron: `*/15 * * * *`
- Меньше нагрузка на сервер

### Каждый час:
- Cron: `0 * * * *`
- Для редких обновлений

### Раз в день (в 3:00):
- Cron: `0 3 * * *`
- Для ночных обновлений
