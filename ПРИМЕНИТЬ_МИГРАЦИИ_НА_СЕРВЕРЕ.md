# Инструкция по применению миграций на сервере

## Проблема

При выполнении `python manage.py migrate catalog` появляется сообщение:
```
Your models in app(s): 'catalog' have changes that are not yet reflected in a migration
```

## Решение

### Шаг 1: Загрузите последние изменения из репозитория

```bash
cd /home/o/onesim8n/onesimus/onesimus
source venv/bin/activate
git pull origin main
```

Это загрузит миграцию `0013_add_farpost_auto_update_fields.py` и другие изменения.

### Шаг 2: Проверьте статус миграций

```bash
python manage.py showmigrations catalog
```

Вы должны увидеть список миграций. Неприменённые будут отмечены как `[ ]`, применённые как `[X]`.

### Шаг 3: Создайте недостающие миграции (если нужно)

Если после git pull всё ещё есть изменения в моделях:

```bash
python manage.py makemigrations catalog
```

Это создаст новые миграции, если есть изменения в моделях, которые не отражены в существующих миграциях.

### Шаг 4: Примените миграции

```bash
python manage.py migrate catalog
```

### Шаг 5: Перезапустите приложение

```bash
touch tmp/restart.txt
```

## Проверка

После применения миграций проверьте:

```bash
python manage.py showmigrations catalog
```

Все миграции должны быть отмечены как `[X]` (применены).

## Если миграция 0013 не загрузилась

Если после `git pull` миграция `0013_add_farpost_auto_update_fields.py` всё ещё отсутствует, проверьте:

```bash
ls -la catalog/migrations/0013_*
```

Если файла нет, возможно нужно:
1. Проверить, что изменения закоммичены локально
2. Отправить изменения в репозиторий: `git push origin main`
3. Затем на сервере: `git pull origin main`
