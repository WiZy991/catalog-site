# Решение проблемы: cron выполняется, но товары не появляются

## 🔴 Проблема

Cron-задача выполняется успешно ("Выполнено"), но товары не появляются на сайте.

## 🔍 Причина

Файлы уже помечены как обработанные (есть маркеры `.processed`), поэтому cron их пропускает:

```
Пропускаем уже обработанный файл: import0_1.xml
Пропускаем уже обработанный файл: offers0_1.xml
```

## ✅ Решение

### Вариант 1: Удалить маркеры и запустить снова (рекомендуется)

Через SSH:

```bash
cd /home/o/onesim8n/onesimus/onesimus/media/1c_exchange
rm -f *.processed
```

Затем запустите cron-задачу снова через интерфейс Beget (кнопка "Запустить").

### Вариант 2: Использовать флаг `--all` для обработки всех файлов

Измените команду в cron на:

```bash
cd /home/o/onesim8n/onesimus/onesimus && /home/o/onesim8n/onesimus/venv/bin/python manage.py process_1c_files --all >> /home/o/onesim8n/onesimus/onesimus/logs/cron_1c_import.log 2>&1
```

Флаг `--all` заставит обработать все файлы, даже если они уже обработаны.

**Примечание:** Используйте `--all` только для теста или если нужно переобработать все файлы. Для постоянной работы лучше использовать вариант 1.

### Вариант 3: Проверить, что файлы действительно новые

Если файлы от 1С не обновляются, cron не будет их обрабатывать повторно (это нормальное поведение).

Для обработки новых файлов:
1. Загрузите новые файлы от 1С в `media/1c_exchange/`
2. Cron автоматически их обработает (если расписание правильное)

## 📊 Ожидаемое поведение

### При первом запуске (файлы не обработаны):
```
Начинаем обработку файлов от 1С...
Найдено файлов: 2
Обработка файла: import0_1.xml...
✓ import0_1.xml: обработано 5862 товаров (создано: 5862, обновлено: 0)
```

### При повторном запуске (файлы уже обработаны):
```
Начинаем обработку файлов от 1С...
Найдено файлов: 2
Пропускаем уже обработанный файл: import0_1.xml
Пропускаем уже обработанный файл: offers0_1.xml
Обработано файлов: 0
```

## ⚠️ Важно

- Маркеры `.processed` создаются автоматически после успешной обработки файла
- Это защита от повторной обработки одних и тех же файлов
- Для обработки новых файлов от 1С маркеры не нужны - они обработаются автоматически
- Если нужно переобработать файлы - удалите маркеры или используйте `--all`

## 🎯 Для автоматической работы

После удаления маркеров и исправления расписания на `*/5 * * * *`:

1. Новые файлы от 1С будут обрабатываться автоматически каждые 5 минут
2. Старые файлы не будут обрабатываться повторно (защита от дублирования)
3. Товары будут добавляться на сайт в течение 5 минут после появления файла
