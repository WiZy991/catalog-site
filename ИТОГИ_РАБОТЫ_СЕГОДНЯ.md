### 5. ✅ Синхронизация остатков: товары без остатка скрываются

**Требование:** Если товара нет на остатке в 1С, он должен быть скрыт на сайте и не попадать на Farpost.

**Реализация:**
- При обработке `offers.xml`: если `quantity = 0` → `is_active = False`
- При обработке `import.xml`: товар создается/обновляется с `is_active = True` только если `quantity > 0`
- На сайте: все запросы товаров фильтруются по `quantity > 0`
- На Farpost: экспортируются только товары с `quantity > 0`
- В поиске, рекомендуемых товарах, sitemap: только товары с остатком

**Важно:** Товары с остатком, но без цены, отображаются везде (цена может быть 0).

**Файлы:**
- `catalog/commerceml_views.py` - обработка остатков
- `catalog/farpost_views.py` - фильтрация при экспорте
- `catalog/views.py` - фильтрация на сайте
- `core/views.py` - рекомендуемые товары
- `catalog/models.py` - подсчет товаров в категориях
- `catalog/sitemaps.py` - sitemap

### 6. ✅ Автоматическое скрытие удаленных товаров из 1С

**Требование:** Если товар удален в 1С, он должен быть скрыт на сайте.

**Реализация:**
- После обработки всех товаров из `import.xml` система сравнивает:
  - Список `external_id` из текущего обмена
  - Товары на сайте с `external_id`, которые были активны
- Товары, которые не пришли в обмене, автоматически скрываются
- Товары не удаляются физически (сохраняется история)
- Устанавливается `is_active = False`, `quantity = 0`, `availability = 'out_of_stock'`

**Логика:**
- Проверяется только по `external_id` (уникальный идентификатор из 1С)
- Товары без `external_id` (созданные вручную) не затрагиваются
- Все действия логируются

**Файлы:** `catalog/commerceml_views.py`

```
    ↓
Сайт (Django)
    ↓
Farpost (экспорт)

Правила синхронизации:
1. quantity > 0 → is_active = True → Видно на сайте и Farpost

4. Цена = 0 → Товар виден, но без цены (если есть остаток)
```

### 7. ✅ Загрузка оптовых товаров из 1С

**Требование:** Товары из 1С должны загружаться не только в розничный, но и в оптовый каталог.

**Реализация:**
- Добавлено автоматическое определение типа каталога по имени файла
- Если в имени файла есть `wholesale`, `опт`, `opt` → товары попадают в оптовый каталог
- Если нет → товары попадают в розничный каталог (по умолчанию)
- Обработка `import.xml` и `offers.xml` работает для обоих типов каталогов
- Синхронизация остатков и удаление товаров работает отдельно для каждого типа каталога
- Оптовые товары фильтруются по остатку (`quantity > 0`)

**Примеры имен файлов:**
- `import.xml` → Розничный каталог
- `import_wholesale.xml` → Оптовый каталог
- `offers.xml` → Цены/остатки для розницы
- `offers_wholesale.xml` → Цены/остатки для опта

**Файлы:**
- `catalog/commerceml_views.py` - обработка CommerceML с определением типа каталога
- `partners/views.py` - фильтрация оптовых товаров по остатку
- `ОПТОВЫЙ_КАТАЛОГ_ИЗ_1С.md` - подробная документация
