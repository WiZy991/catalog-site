# Диагностика проблемы со статикой

## Проблема
Оба endpoint (`/static/css/style.css` и `/test-static-css/`) возвращают 404.

## Шаги диагностики на сервере

### 1. Проверьте, что файл `config/urls.py` обновлен на сервере:

```bash
cd ~/onesimus/onesimus
git status
git pull
cat config/urls.py | grep -A 5 "test_simple"
```

Должна быть функция `test_simple` и путь `test-simple/`.

### 2. Проверьте простой тестовый endpoint:

```bash
curl https://onesimus25.ru/test-simple/
```

Если возвращает "Test endpoint works!" - Django работает, проблема в статике.
Если возвращает 404 - Django не видит новый код.

### 3. Если `test-simple/` не работает, проверьте логи Django:

```bash
tail -50 logs/django.log
# или
tail -50 ~/logs/django.log
# или
journalctl -u passenger -n 50
```

Ищите ошибки типа:
- `ModuleNotFoundError`
- `ImportError`
- `SyntaxError`
- `TemplateSyntaxError`

### 4. Принудительно перезапустите Django:

```bash
# Удалите файл перезапуска и создайте заново
rm -f tmp/restart.txt
touch tmp/restart.txt

# Подождите 10-20 секунд
sleep 15

# Проверьте снова
curl https://onesimus25.ru/test-simple/
```

### 5. Проверьте, что файл существует:

```bash
ls -la staticfiles/css/style.css
cat staticfiles/css/style.css | head -5
```

### 6. Проверьте настройки STATIC_ROOT:

```bash
python manage.py shell -c "from django.conf import settings; print(settings.STATIC_ROOT)"
```

Должен вывести путь к `staticfiles/`.

### 7. Если все еще не работает, временно верните DEBUG=True:

```bash
sed -i "s/DEBUG = False/DEBUG = True/" config/settings.py
touch tmp/restart.txt
sleep 15
curl https://onesimus25.ru/static/css/style.css
```

Если с `DEBUG=True` работает - проблема в том, что при `DEBUG=False` Django не раздает статику через `static()` в urls.py, даже если мы убрали условие.

## Решение

Если `test-simple/` работает, но статика нет - значит Nginx блокирует `/static/`.

В этом случае нужно:
1. Настроить Nginx через панель Beget
2. Или изменить `STATIC_URL` на другой путь (например, `/assets/`)
