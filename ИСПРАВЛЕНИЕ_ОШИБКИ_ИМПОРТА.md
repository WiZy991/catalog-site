# Исправление ошибки импорта товаров

## Проблема

При импорте товаров из 1С возникает ошибка:
```
no such table: catalog_productcharacteristic
TransactionManagementError: An error occurred in the current transaction
```

## Причина

На сервере не применены миграции 0012 и 0013, которые создают таблицу `catalog_productcharacteristic`.

## Решение

### 1. Применить миграции на сервере (ОБЯЗАТЕЛЬНО):

```bash
cd /home/o/onesim8n/onesimus/onesimus
source venv/bin/activate
git pull origin main
python manage.py migrate catalog
touch tmp/restart.txt
```

### 2. Код уже исправлен

Добавлена обработка ошибок в коде, чтобы если таблицы нет, код не падал полностью, а просто пропускал создание характеристик (они всё равно сохраняются в поле `product.characteristics`).

**Файлы с исправлениями:**
- `catalog/commerceml_views.py` - обработка ошибок при работе с ProductCharacteristic
- `catalog/one_c_views.py` - обработка ошибок при работе с ProductCharacteristic

### 3. После применения миграций

После применения миграций импорт должен работать нормально, и характеристики будут создаваться в таблице `ProductCharacteristic`.

## Проверка

После применения миграций проверьте:

```bash
python manage.py showmigrations catalog
```

Все миграции должны быть отмечены как `[X]` (применены).
