# Поиск процесса Django сервера

## Расширенный поиск процессов

Выполните на сервере:

```bash
# Найти все процессы Python
ps aux | grep python | grep -v grep

# Найти процессы, связанные с Django/WSGI
ps aux | grep -E "django|wsgi|beget" | grep -v grep

# Найти процессы на портах (обычно 8000, 8080, или другие)
netstat -tlnp 2>/dev/null | grep -E ":80|:8000|:8080"
# или
ss -tlnp | grep -E ":80|:8000|:8080"

# Проверить, есть ли supervisor
which supervisorctl
supervisorctl status

# Проверить systemd (если доступен)
systemctl --user list-units | grep django
systemctl list-units | grep django
```

## Если сервер запущен через beget

На beget обычно Django запускается автоматически через их систему управления. Проверьте:

1. Зайдите в панель beget: https://cp.beget.com
2. Найдите ваш сайт
3. Перейдите в раздел "Управление сайтом" или "Процессы"
4. Там должен быть процесс Python/Django
5. Нажмите "Перезапустить"

## Альтернативный способ - проверить через логи

```bash
# Проверить логи beget
ls -la ~/logs/
tail -f ~/logs/*.log

# Проверить логи Django (если есть)
ls -la ~/onesimus/onesimus/logs/
```

## Если ничего не найдено

Возможно, сервер запускается автоматически при обращении к сайту (через CGI/FastCGI). В этом случае:

1. Просто обратитесь к URL - сервер запустится автоматически
2. Или перезапустите через beget панель

## Проверка через веб-интерфейс

После перезапуска через beget панель, проверьте URL напрямую в браузере или через curl.
